# Muse Design Principles

## 1. Function First

Every interface element must have a clear purpose.

If a component does not improve the experience, it should not exist.

---

## 2. Touch First

Muse is designed for fingers, not for a mouse.

Every interactive element must be easy to press.

---

## 3. Premium Simplicity

Muse should feel elegant without being complicated.

No unnecessary effects.

No visual clutter.

---

## 4. Instant Understanding

A user should understand every screen within a few seconds.

The interface should require almost no explanation.

---

## 5. Fluidity

Animations should never attract attention.

They exist only to make interactions feel natural.

---

## 6. Consistency

Every screen must look like it belongs to the same product.

Spacing, typography, colors and interactions must remain consistent.

---

## 7. Product Before Technology

Muse should feel like a real household product, not a web application.
